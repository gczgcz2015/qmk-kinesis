# 接线说明：Joy-Con WASD 分支

本分支把原来的左右拇指区矩阵停用，左侧拇指区改为 1 个 Joy-Con 摇杆。
摇杆方向在固件里硬编码为 `W/A/S/D`，支持斜向同时触发两个键。

完整图见 [wiring-layout.svg](wiring-layout.svg)。

## 材料

- Waveshare RP2040-Zero × 2
- MX 兼容开关 × 64
- 1N4148 穿孔二极管 × 64
- 三芯 TRS 插座 × 2
- 三芯 TRS 公对公线 × 1
- Joy-Con 摇杆 × 1
- Joy-Con 5P FPC 转接板 × 1
- 26–28 AWG 单芯线：矩阵行列
- 24–28 AWG 多芯软线：控制器、TRS、Joy-Con

建议先完成并用万用表检查主键区矩阵，再接控制器、TRS 和 Joy-Con。

## 总体结构

每侧主键区组成一个 5 行矩阵。QMK 仍保留原 `LAYOUT_5x7` 的 12×7 全局矩阵形状，
但本分支把左右拇指行 `R5` 设为 `NO_PIN`，Vial/VIA 中也隐藏这些位置。

```text
电脑 USB
   │
   ▼
左 RP2040-Zero ─── TRS：DATA + 5V + GND ─── 右 RP2040-Zero
   │                                              │
左主键矩阵 R0-R4                             右主键矩阵 R0-R4
   │
左 Joy-Con：X/Y 模拟输入 → WASD
```

左侧固定为 USB 主控。右侧正常使用时不连接 USB。

## RP2040-Zero 引脚

两侧矩阵和 TRS 接法相同：

| 功能 | RP2040-Zero |
|---|---|
| TRS 半双工数据 | GP0 |
| 列 C0 | GP2 |
| 列 C1 | GP3 |
| 列 C2 | GP4 |
| 列 C3 | GP5 |
| 列 C4 | GP6 |
| 列 C5 | GP7 |
| 列 C6 | GP8 |
| 行 R0 | GP14 |
| 行 R1 | GP15 |
| 行 R2 | GP26 |
| 行 R3 | GP27 |
| 行 R4 | GP9 |
| 行 R5 / 原拇指区 | 不接，固件中为 `NO_PIN` |
| TRS 电源 | 5V |
| TRS 地 | GND |

左侧 Joy-Con 额外接法：

| Joy-Con / FPC 转接板 | 左 RP2040-Zero |
|---|---|
| VCC | 3V3 |
| GND | GND |
| X / VRX | GP28 / ADC2 |
| Y / VRY | GP29 / ADC3 |
| SW / BTN | 暂不接 |

不要把 Joy-Con 的 VCC 接 5V。RP2040 的 ADC 输入只能按 3.3V 逻辑使用。

GP1 保留。板载 GP16 WS2812 不由本固件使用。

## 行列编号

从佩戴者视角俯视键盘：

- 行 `R0..R4`：从上到下五排主键区。
- 行 `R5`：原拇指区，当前分支停用，不安装开关，不接矩阵。
- 每侧都从最外侧小指边向中央编号 `C0..C6`。

左右列号是镜像的：

```text
左半边（外 → 内）               右半边（内 ← 外）
C0 C1 C2 C3 C4 C5 C6     C6 C5 C4 C3 C2 C1 C0
```

当前实体布局使用下列主键交点；`—` 是 QMK 保留但不安装开关、不接线的位置：

```text
       C0 C1 C2 C3 C4 C5 C6
R0     ○  ○  ○  ○  ○  ○  ○
R1     ○  ○  ○  ○  ○  ○  ○
R2     ○  ○  ○  ○  ○  ○  ○
R3     ○  ○  ○  ○  ○  ○  —
R4     ○  ○  ○  ○  ○  —  —
R5     —  —  —  —  —  —  —   原拇指区停用
```

QMK 的完整矩阵定义仍保留 84 个坐标，当前实体键盘和 Vial/VIA 使用 64 个。
以下 20 个位置无需安装开关或接线，并在 Vial/VIA 中隐藏：

| 左侧本地位置 | Vial 全局位置 | 右侧本地位置 | Vial 全局位置 |
|---|---|---|---|
| R3 C6 | 3,6 | R3 C6 | 9,6 |
| R4 C5 | 4,5 | R4 C5 | 10,5 |
| R4 C6 | 4,6 | R4 C6 | 10,6 |
| R5 C0–C6 | 5,0–5,6 | R5 C0–C6 | 11,0–11,6 |

中央三键竖列从上到下使用 `R0 C6`、`R1 C6`、`R2 C6`。它不是额外矩阵；
下面的 `R3 C6`、`R4 C6` 是保留但不接线的位置。

## 单个开关和二极管

固件使用 `COL2ROW`。每颗二极管的黑色带环端必须朝行线：

```text
列线 Cn ── 开关 ── 二极管无环端 ──|带环端|── 行线 Rn
```

具体步骤：

1. 每个开关的一只焊脚连接所属列线。
2. 另一只焊脚连接 1N4148 的无环端。
3. 1N4148 的黑色带环端连接所属行线。
4. 同一列的开关焊脚相连，同一行的二极管带环端相连。
5. 行线和列线交叉处必须绝缘，不能相碰。

焊完每一行后，用二极管档确认所有二极管方向一致。

## Joy-Con 行为

固件读取左侧 `GP28` 和 `GP29`：

- X 轴左推：`A`
- X 轴右推：`D`
- Y 轴上推：`W`
- Y 轴下推：`S`
- 斜向：同时按两个键，例如左上 = `A + W`

每次上电时请先松开摇杆并保持静止约 1 秒。固件会：

- 等待 `250 ms`，避免把刚上电时的 ADC 波动当作中心。
- 连续采集 `32` 个稳定样本，分别计算 X/Y 实际中心值。
- 如果任一轴的样本跨度超过 `24`，丢弃这一组并自动重新采集。
- 如果计算出的中心不在 `256–768`，按未接好或未松开摇杆处理并继续重试。
- 用二分之一低通滤波抑制 ADC 抖动。
- 偏离中心 `110` 时按下方向键，回到 `70` 内时释放，形成滞回区。

代码中的 `512` 只是在校准完成前使用的安全初值；校准期间不会发送 `W/A/S/D`。
固件不会在使用过程中自动追踪中心，避免长时间推杆时把有效方向误学成新中心。

如果方向反了，优先改 `keymap.c` 的轴方向逻辑；不要通过把电源改到 5V 解决。

## TRS 左右连接

两侧 TRS 插座必须一一对应，不交叉：

| TRS 触点 | 左侧 | 右侧 |
|---|---|---|
| Tip（尖端） | GP0 | GP0 |
| Ring（中环） | 5V | 5V |
| Sleeve（尾套） | GND | GND |

这是 RP2040 PIO 半双工连接，所以只有一根数据线，不需要 TX/RX 交叉，也不需要
外接上拉电阻。

### TRS 警告

TRS 插头插入和拔出时，触点会短暂擦过其他触点，可能把 5V 瞬间接到数据或地。

必须遵守：

1. 先拔掉左侧 USB。
2. 确认右侧也未连接 USB。
3. 再插入或拔出 TRS。
4. 插好 TRS 后，只给左侧接 USB。

如果以后要求带电热插拔，应改用不会在插拔过程中短接的专用连接器；仅修改固件不能
消除 TRS 的物理风险。

## 上电前检查

万用表检查：

1. 左右两侧 5V 与 GND 不短路。
2. 每条行线之间不短路。
3. 每条列线之间不短路。
4. 行线与列线在未按键时不导通。
5. 按键时，仅对应行列通过二极管单向导通。
6. TRS 两端 Tip、Ring、Sleeve 分别直通。
7. TRS 任意两个触点之间不短路。
8. 左侧 Joy-Con：3V3 与 GND 不短路；GP28/GP29 不应与 5V、GND 短路。

## 首次调试顺序

1. 不接 TRS，分别给两块 RP2040-Zero 刷入同一个 UF2。
2. 左右主键区确认 `R4` 已经改接 `GP9`，原 `GP28` 不再接矩阵。
3. 两侧断电后接 TRS。
4. 只给左侧接 USB。
5. 先在普通键盘测试工具中逐键确认 64 个主键矩阵。
6. 再接左侧 Joy-Con；每次上电先松开摇杆约 1 秒，再测试 `W/A/S/D` 和斜向组合。
7. 使用 Chrome/Edge 打开 Vial Web，授权设备后确认自动显示 64 个主键。
8. 若整行失效，检查对应 `R` 引脚；若整列失效，检查对应 `C` 引脚。
9. 若右侧全部失效，依次检查 TRS 的 GND、5V、GP0 数据和插头是否完全插入。
