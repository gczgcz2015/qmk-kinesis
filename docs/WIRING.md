# 接线说明

## 材料

- Waveshare RP2040-Zero × 2
- MX 兼容开关 × 84
- 1N4148 穿孔二极管 × 84
- 三芯 TRS 插座 × 2
- 三芯 TRS 公对公线 × 1
- 26–28 AWG 单芯线：矩阵行列
- 24–28 AWG 多芯软线：控制器和 TRS

建议先完成并用万用表检查矩阵，再接控制器和 TRS。

每个可见键的第一层键位、矩阵坐标、GPIO 引脚对、行列总线接法和二极管方向见
[wiring-layout.svg](wiring-layout.svg)。

## 总体结构

每侧独立组成一个 6×7 矩阵，使用相同 GPIO。QMK 会把右侧本地行
`R0..R5` 映射成全局矩阵行 `6..11`。

```text
电脑 USB
   │
   ▼
左 RP2040-Zero ─── TRS：DATA + 5V + GND ─── 右 RP2040-Zero
   │                                              │
左 6×7 矩阵                                  右 6×7 矩阵
```

左侧固定为 USB 主控。右侧正常使用时不连接 USB。

## RP2040-Zero 引脚

两侧接法完全相同：

| 功能 | RP2040-Zero |
|---|---|
| TRS 半双工数据 | GP0 |
| 列 C0 | GP2 |
| 列 C1 | GP3 |
| 列 C2 | GP4 |
| 列 C3 | GP5 |
| 列 C4 | GP6 |
| 列 C5 | GP7 |
| 列 C6 | GP8 |
| 行 R0 | GP14 |
| 行 R1 | GP15 |
| 行 R2 | GP26 |
| 行 R3 | GP27 |
| 行 R4 | GP28 |
| 行 R5 / 拇指区 | GP29 |
| TRS 电源 | 5V |
| TRS 地 | GND |

GP1 保留。板载 GP16 WS2812 不由本固件使用。

## 行列编号

从佩戴者视角俯视键盘：

- 行 `R0..R4`：从上到下五排主键区。
- 行 `R5`：该侧七个拇指/扩展矩阵位置，其中 `C3` 在 VIA 中隐藏。
- 每侧都从最外侧小指边向中央编号 `C0..C6`。

因此整体俯视时，左右列号是镜像的：

```text
左半边（外 → 内）               右半边（内 ← 外）
C0 C1 C2 C3 C4 C5 C6     C6 C5 C4 C3 C2 C1 C0
```

主键区每个交点都接开关：

```text
       C0 C1 C2 C3 C4 C5 C6
R0     ○  ○  ○  ○  ○  ○  ○
R1     ○  ○  ○  ○  ○  ○  ○
R2     ○  ○  ○  ○  ○  ○  ○
R3     ○  ○  ○  ○  ○  ○  ○
R4     ○  ○  ○  ○  ○  ○  ○
R5     ○  ○  ○  ○  ○  ○  ○   拇指/扩展区
```

实体上仍接满 84 键。VIA 只隐藏八个位置，不影响矩阵接线：

| 左侧本地位置 | VIA 全局位置 | 右侧本地位置 | VIA 全局位置 |
|---|---|---|---|
| R0 C6 | 0,6 | R0 C6 | 6,6 |
| R4 C5 | 4,5 | R4 C5 | 10,5 |
| R4 C6 | 4,6 | R4 C6 | 10,6 |
| R5 C3 | 5,3 | R5 C3 | 11,3 |

中央三键竖列使用 `R1 C6`、`R2 C6`、`R3 C6`。它不是额外矩阵；
上下两端的 `R0 C6`、`R4 C6` 是隐藏位置。

### 拇指区矩阵位置

下面是当前默认键位对应的本地矩阵坐标。键位可以在 VIA 中修改，但焊接坐标不会
随键位变化。

```text
左侧拇指区

                 C0 Ctrl        C2 Alt
C4 Backspace 2u  C6 Delete 2u   C1 Home
                                 C5 End

右侧拇指区

C4 GUI/Win       C2 Ctrl
C6 Page Up       C5 Enter 2u    C0 Space 2u
C1 Page Down
```

`C3` 是每侧保留但在 VIA 中隐藏的第七个拇指/扩展位置。

VIA 中的纵向 2u 只表示键帽外观。Backspace、Delete、Enter、Space 每个仍然只
使用一个 MX 开关、一个二极管和一个 `R5/Cn` 交点，不能把两个矩阵交点并联。
如果实体键盘也安装 2u 键帽，外壳/定位板必须提供对应的单开关孔位，并使用兼容的
2u POS 键帽或稳定器；修改 VIA JSON 不会改变外壳结构。

## 单个开关和二极管

固件使用 `COL2ROW`。每颗二极管的黑色带环端必须朝行线：

```text
列线 Cn ── 开关 ── 二极管无环端 ──|带环端|── 行线 Rn
```

具体步骤：

1. 每个开关的一只焊脚连接所属列线。
2. 另一只焊脚连接 1N4148 的无环端。
3. 1N4148 的黑色带环端连接所属行线。
4. 同一列的开关焊脚相连，同一行的二极管带环端相连。
5. 行线和列线交叉处必须绝缘，不能相碰。

焊完每一行后，用二极管档确认所有二极管方向一致。

## TRS 左右连接

两侧 TRS 插座必须一一对应，不交叉：

| TRS 触点 | 左侧 | 右侧 |
|---|---|---|
| Tip（尖端） | GP0 | GP0 |
| Ring（中环） | 5V | 5V |
| Sleeve（尾套） | GND | GND |

这是 RP2040 PIO 半双工连接，所以只有一根数据线，不需要 TX/RX 交叉，也不需要
外接上拉电阻。

### TRS 警告

TRS 插头插入和拔出时，触点会短暂擦过其他触点，可能把 5V 瞬间接到数据或地。

必须遵守：

1. 先拔掉左侧 USB。
2. 确认右侧也未连接 USB。
3. 再插入或拔出 TRS。
4. 插好 TRS 后，只给左侧接 USB。

如果以后要求带电热插拔，应改用不会在插拔过程中短接的专用连接器；仅修改固件不能
消除 TRS 的物理风险。

## 上电前检查

万用表检查：

1. 左右两侧 5V 与 GND 不短路。
2. 每条行线之间不短路。
3. 每条列线之间不短路。
4. 行线与列线在未按键时不导通。
5. 按键时，仅对应行列通过二极管单向导通。
6. TRS 两端 Tip、Ring、Sleeve 分别直通。
7. TRS 任意两个触点之间不短路。

## 首次调试顺序

1. 不接 TRS，分别给两块 RP2040-Zero 刷入同一个 UF2。
2. 两侧断电后接 TRS。
3. 只给左侧接 USB。
4. 先在普通键盘测试工具中逐键确认矩阵。
5. 再加载 VIA JSON，确认显示 76 键并测试四层。
6. 若整行失效，检查对应 `R` 引脚；若整列失效，检查对应 `C` 引脚。
7. 若右侧全部失效，依次检查 TRS 的 GND、5V、GP0 数据和插头是否完全插入。
