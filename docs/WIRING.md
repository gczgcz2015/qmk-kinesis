# 接线说明：Ogen Lite PMW3360 轨迹球测试分支

本分支把原来的左右拇指区矩阵停用，左侧拇指区改为 1 个
[Ogen Lite V1.3](https://github.com/ghostlybutterfly/Ogen) / PMW3360 轨迹球。
测试固件只验证光标 X/Y 移动，不配置鼠标按键或滚轮。

完整图见 [wiring-layout.svg](wiring-layout.svg)。

## 材料

- Waveshare RP2040-Zero × 2
- MX 兼容开关 × 64
- 1N4148 穿孔二极管 × 64
- 三芯 TRS 插座 × 2
- 三芯 TRS 公对公线 × 1
- Ogen Lite V1.3 / PMW3360 模块 × 1
- 与外壳匹配的轨迹球和支撑结构 × 1
- 26–28 AWG 单芯线：矩阵行列
- 24–28 AWG 多芯软线：控制器、TRS、Ogen Lite

建议先完成并用万用表检查主键区矩阵，再接控制器、TRS 和轨迹球。

## 总体结构

每侧主键区组成一个 5 行矩阵。QMK 仍保留原 `LAYOUT_5x7` 的 12×7 全局矩阵形状，
但本分支把左右拇指行 `R5` 设为 `NO_PIN`，Vial/VIA 中也隐藏这些位置。

```text
电脑 USB
   │
   ▼
左 RP2040-Zero ─── TRS：DATA + 5V + GND ─── 右 RP2040-Zero
   │                                              │
左主键矩阵 R0-R4                             右主键矩阵 R0-R4
   │
左 Ogen Lite：PMW3360 SPI → USB 鼠标光标
```

左侧固定为 USB 主控。右侧正常使用时不连接 USB。

## RP2040-Zero 引脚

两侧矩阵和 TRS 接法相同：

| 功能 | RP2040-Zero |
|---|---|
| TRS 半双工数据 | GP0 |
| 列 C0 | GP2 |
| 列 C1 | GP3 |
| 列 C2 | GP4 |
| 列 C3 | GP5 |
| 列 C4 | GP6 |
| 列 C5 | GP7 |
| 列 C6 | GP8 |
| 行 R0 | GP14 |
| 行 R1 | GP15 |
| 行 R2 | GP26 |
| 行 R3 | GP27 |
| 行 R4 | GP9 |
| 行 R5 / 原拇指区 | 不接，固件中为 `NO_PIN` |
| TRS 电源 | 5V |
| TRS 地 | GND |

左侧 Ogen Lite V1.3 额外接法：

| Ogen Lite 丝印 | 左 RP2040-Zero |
|---|---|
| VCC | 3V3 |
| GND | GND |
| SCL | GP10 / SPI1 SCK |
| MOS | GP11 / SPI1 MOSI |
| MIS | GP12 / SPI1 MISO |
| SS | GP13 / SPI1 CS |
| MOT | 不接 |
| RES | 不接 |

不要把 Ogen Lite 的 VCC 接 5V。Lite 版没有电平转换，只能使用 3.3V 电源和逻辑。
`MOT` 不是最小轮询方案所需的中断脚；`RES` 也不需要连接，QMK 驱动通过 SPI
执行上电复位和固件上传。

GP1、GP28、GP29 保留。板载 GP16 WS2812 不由本固件使用。

## 行列编号

从佩戴者视角俯视键盘：

- 行 `R0..R4`：从上到下五排主键区。
- 行 `R5`：原拇指区，当前分支停用，不安装开关，不接矩阵。
- 每侧都从最外侧小指边向中央编号 `C0..C6`。

左右列号是镜像的：

```text
左半边（外 → 内）               右半边（内 ← 外）
C0 C1 C2 C3 C4 C5 C6     C6 C5 C4 C3 C2 C1 C0
```

当前实体布局使用下列主键交点；`—` 是 QMK 保留但不安装开关、不接线的位置：

```text
       C0 C1 C2 C3 C4 C5 C6
R0     ○  ○  ○  ○  ○  ○  ○
R1     ○  ○  ○  ○  ○  ○  ○
R2     ○  ○  ○  ○  ○  ○  ○
R3     ○  ○  ○  ○  ○  ○  —
R4     ○  ○  ○  ○  ○  —  —
R5     —  —  —  —  —  —  —   原拇指区停用
```

QMK 的完整矩阵定义仍保留 84 个坐标，当前实体键盘和 Vial/VIA 使用 64 个。
以下 20 个位置无需安装开关或接线，并在 Vial/VIA 中隐藏：

| 左侧本地位置 | Vial 全局位置 | 右侧本地位置 | Vial 全局位置 |
|---|---|---|---|
| R3 C6 | 3,6 | R3 C6 | 9,6 |
| R4 C5 | 4,5 | R4 C5 | 10,5 |
| R4 C6 | 4,6 | R4 C6 | 10,6 |
| R5 C0–C6 | 5,0–5,6 | R5 C0–C6 | 11,0–11,6 |

中央三键竖列从上到下使用 `R0 C6`、`R1 C6`、`R2 C6`。它不是额外矩阵；
下面的 `R3 C6`、`R4 C6` 是保留但不接线的位置。

## 单个开关和二极管

固件使用 `COL2ROW`。每颗二极管的黑色带环端必须朝行线：

```text
列线 Cn ── 开关 ── 二极管无环端 ──|带环端|── 行线 Rn
```

具体步骤：

1. 每个开关的一只焊脚连接所属列线。
2. 另一只焊脚连接 1N4148 的无环端。
3. 1N4148 的黑色带环端连接所属行线。
4. 同一列的开关焊脚相连，同一行的二极管带环端相连。
5. 行线和列线交叉处必须绝缘，不能相碰。

焊完每一行后，用二极管档确认所有二极管方向一致。

## 轨迹球行为

固件使用 Vial-QMK 原生 `pmw3360` pointing-device 驱动读取左侧 SPI1：

- 默认灵敏度：`1600 CPI`
- 只输出光标 X/Y 移动
- 不配置鼠标左/中/右键
- 不配置滚轮或自动鼠标层

建议让 Ogen PCB 丝印的 `UP` 箭头朝向键盘顶部。若光标方向和实际滚球方向不一致，
再通过 `POINTING_DEVICE_ROTATION_*` 或 `POINTING_DEVICE_INVERT_*` 宏校正；第一次测试
先保持默认方向，便于确认 SPI 通信本身正常。

PMW3360 上电后由 QMK 上传传感器 SROM，无需摇杆式中心校准。无论方向是否正确，
都不要通过改接 5V 处理。

## TRS 左右连接

两侧 TRS 插座必须一一对应，不交叉：

| TRS 触点 | 左侧 | 右侧 |
|---|---|---|
| Tip（尖端） | GP0 | GP0 |
| Ring（中环） | 5V | 5V |
| Sleeve（尾套） | GND | GND |

这是 RP2040 PIO 半双工连接，所以只有一根数据线，不需要 TX/RX 交叉，也不需要
外接上拉电阻。

### TRS 警告

TRS 插头插入和拔出时，触点会短暂擦过其他触点，可能把 5V 瞬间接到数据或地。

必须遵守：

1. 先拔掉左侧 USB。
2. 确认右侧也未连接 USB。
3. 再插入或拔出 TRS。
4. 插好 TRS 后，只给左侧接 USB。

如果以后要求带电热插拔，应改用不会在插拔过程中短接的专用连接器；仅修改固件不能
消除 TRS 的物理风险。

## 上电前检查

万用表检查：

1. 左右两侧 5V 与 GND 不短路。
2. 每条行线之间不短路。
3. 每条列线之间不短路。
4. 行线与列线在未按键时不导通。
5. 按键时，仅对应行列通过二极管单向导通。
6. TRS 两端 Tip、Ring、Sleeve 分别直通。
7. TRS 任意两个触点之间不短路。
8. 左侧 Ogen Lite：3V3 与 GND 不短路；GP10–GP13 不应与 5V、GND 短路。

## 首次调试顺序

1. 不接 TRS，分别给两块 RP2040-Zero 刷入同一个 UF2。
2. 左右主键区确认 `R4` 接 `GP9`；轨迹球使用的 GP10–GP13 不接矩阵。
3. 两侧断电后接 TRS。
4. 只给左侧接 USB。
5. 先在普通键盘测试工具中逐键确认 64 个主键矩阵。
6. 再接左侧 Ogen Lite；上电后滚动轨迹球，确认系统光标能移动。
7. 使用 Chrome/Edge 打开 Vial Web，授权设备后确认自动显示 64 个主键。
8. 若整行失效，检查对应 `R` 引脚；若整列失效，检查对应 `C` 引脚。
9. 若轨迹球无响应，先断电，再逐一检查 3V3、GND、GP10–GP13 及 Ogen 丝印方向。
10. 若右侧全部失效，依次检查 TRS 的 GND、5V、GP0 数据和插头是否完全插入。
